# Clipper — Chrome Extension v1.5.1

Chrome-расширение для извлечения контента веб-страниц и конвертации в Markdown. Отсекает рекламу, навигацию, футеры и прочий мусор, оставляя только полезное содержимое. Работает с обычными статьями, SPA, Twitter/X, каталогами и комментариями.

## Как это работает

Расширение использует многоэтапный пайплайн извлечения:

1. **Очистка DOM** — удаление cookie-баннеров, сайдбаров, навигации, рекламы, кнопок шаринга, auth-заглушек по селекторам и ключевым фразам
2. **Mozilla Readability** — тот же алгоритм, что в Reader View Firefox; определяет основной контент страницы
3. **Дополнительный контент** — поиск комментариев (30+ селекторов) и похожих материалов за пределами основной статьи
4. **Fallback-экстракторы** — SPA-виджеты (`__NEXT_DATA__`, JSON state), каталоги (grid-секции с повторяющимися карточками), плоский сбор текста
5. **Twitter/X** — отдельный экстрактор, собирающий твиты, треды и ленту напрямую из DOM
6. **Turndown + GFM** — конвертация очищенного HTML в GitHub Flavored Markdown (таблицы, зачёркивания, списки задач)

Content script загружается **только по клику** на иконку расширения (ленивая инъекция), а не на каждой открытой вкладке. Сборка — **esbuild**, код модульный (ES modules).

## Возможности

- Извлечение контента любой веб-страницы в Markdown одним кликом
- YAML frontmatter: title, source, created, description, tags
- Многоуровневая очистка: реклама, навигация, футеры, сайдбары, cookie-баннеры, кнопки шаринга
- Сохранение в `.md` файл или копирование в буфер обмена
- Twitter/X — твиты, треды, лента, quote-твиты как цитаты
- SPA-страницы (Next.js, Kayak и др.) — извлечение из `__NEXT_DATA__` и JSON state
- Каталоги (Amazon, интернет-магазины) — извлечение карточек товаров
- Комментарии (Hacker News и др.) — извлечение обсуждений
- GitHub Flavored Markdown — таблицы, зачёркивания, списки задач

## Установка

1. Клонируйте репозиторий
2. Откройте `chrome://extensions` в Chrome
3. Включите **Режим разработчика**
4. Нажмите **Загрузить распакованное расширение**
5. Выберите папку репозитория

`dist/` уже содержит собранные файлы — `npm run build` не обязателен.

## Разработка

```bash
npm install
npm run build    # esbuild → dist/content.js + dist/popup.js
npm run watch    # watch-режим
```

После изменений в `src/` выполните `npm run build` и перезагрузите расширение в `chrome://extensions`.

### Тестирование

`tests/fixtures-urls.txt` содержит набор эталонных URL для ручного регрессионного тестирования. Для каждого URL сохраните ожидаемый Markdown в `tests/baseline/<slug>.md`.

## Использование

1. Откройте веб-страницу или x.com
2. Нажмите иконку **Clipper**
3. Дождитесь статуса «Готово: {заголовок}»
4. **Скачать .md** или **Копировать в буфер**

## Структура проекта

```
├── manifest.json
├── popup.html
├── dist/
│   ├── content.js          # собранный content script
│   └── popup.js            # собранный popup
├── src/
│   ├── clipper/
│   │   ├── index.js        # точка входа content script
│   │   ├── dom-cleanup.js  # удаление мусора из DOM
│   │   ├── extract-document.js  # пайплайн извлечения
│   │   ├── extract-twitter.js   # экстрактор X/Twitter
│   │   ├── extra-content.js     # комментарии, похожее
│   │   ├── frontmatter.js       # YAML frontmatter
│   │   ├── turndown-config.js   # конфигурация HTML→Markdown
│   │   ├── html-utils.js        # утилиты
│   │   └── validators.js        # валидация секций
│   └── popup/
│       └── popup.js         # логика popup-интерфейса
├── build.mjs               # esbuild-сборка
├── package.json
├── tests/
│   ├── fixtures-urls.txt
│   └── baseline/
└── icons/
```

## Разрешения

| Разрешение | Зачем |
|------------|-------|
| `activeTab` | Доступ к активной вкладке |
| `scripting` | Инжект content script по клику |
| `downloads` | Скачивание `.md` файла |
| `clipboardWrite` | Копирование в буфер обмена |
